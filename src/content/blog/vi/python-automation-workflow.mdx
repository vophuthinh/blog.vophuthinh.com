Automation chạy ổn ở máy local chưa đủ. Mục tiêu của workflow doanh nghiệp là **độ ổn định**, **khả năng quan sát**, và **khả năng mở rộng**.

## Kiến trúc đề xuất

- Layer `ingestion`: nhận dữ liệu đầu vào.
- Layer `processing`: xử lý nghiệp vụ.
- Layer `delivery`: xuất kết quả.
- Layer `monitoring`: log, metrics, alert.

## Skeleton job runner

```python
from datetime import datetime

def run_job(payload: dict) -> dict:
    started_at = datetime.utcnow()
    result = {
        "status": "success",
        "processed_records": len(payload.get("records", [])),
        "started_at": started_at.isoformat(),
    }
    return result
```

## Checklist production

- Idempotency key cho mỗi lần chạy.
- Retry với exponential backoff.
- Alert khi SLA vượt ngưỡng.

Workflow tốt không chỉ chạy được, mà còn phải giúp team vận hành tự tin khi sự cố xảy ra.
