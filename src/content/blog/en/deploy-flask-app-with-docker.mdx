Docker standardizes deployment and reduces the gap between development and production environments.

## Reference Dockerfile

```dockerfile
FROM python:3.12-slim

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .
EXPOSE 8000
CMD ["gunicorn", "app:app", "--bind", "0.0.0.0:8000"]
```

## Production notes

- Keep images small for faster build and pull times.
- Avoid running containers as root.
- Add healthchecks for automated recovery.

Combined with CI/CD, this approach creates a faster and safer release pipeline.
